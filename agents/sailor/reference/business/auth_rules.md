# 登录注册功能业务规则

## 一、业务功能点

| 功能编号 | 功能名称 | 说明 |
|---------|---------|------|
| F001 | 账号登录 | 管理员/用户使用邮箱+密码登录系统 |
| F002 | 找回密码 | 用户通过邮箱+验证码重置密码 |
| F003 | 图形验证码 | 登录失败5次后触发图形验证码 |
| F004 | 发送邮箱验证码 | 找回密码时向邮箱发送4位数字验证码 |
| F005 | 密码强度校验 | 密码8-16位，含大小写字母、数字、特殊字符 |

## 四、API 文档

### 4.1 用户登录（Code 换 Token）

- **URL**: `/auth/login`
- **Method**: POST
- **Request**:
```json
{
  "email": "user@example.com",
  "password": "Password123!",
  "code": "abc123",
  "key": "captcha_key"
}
```

**业务逻辑**:
1. 登录失败5次以内，返回错误码 `100009` 或 `100010`
2. 失败5次后，前端需显示图形验证码
3. 密码校验: `md5(salt + "+_)(*&qwer" + password)`
4. 成功登录返回 JWT Token，有效期10天

### 4.2 找回密码

- **URL**: `/auth/find-password`
- **Method**: POST
- **Request**: `{ "email", "code", "password" }`

**业务逻辑**:
1. 验证码5分钟内有效，验证后删除
2. 密码格式校验: 8-16位，含大小写字母、数字、特殊字符
3. 密码更新后 salt 重新生成

### 4.3 发送邮箱验证码

- **URL**: `/auth/send-mail-code`
- **Method**: POST
- **Request**: `{ "email", "code", "key" }`

### 4.4 获取图形验证码

- **URL**: `/auth/captcha`
- **Method**: GET
- **Response**: `{ "key": "xxx", "img": "data:image/png;base64,..." }`

## 五、JWT Token 规范

### 5.1 PHP 后端 Token

**JWT Key**: `abcdefghijklmna`

**Payload**:
```json
{
  "i": 123,           // 用户ID
  "r": 0,             // 角色: 0企业/1超管/2国级/3省级
  "ex": 1767225600    // 过期时间戳(10天后)
}
```

### 5.2 Go 后端 Token (优企平台)

**JWT Key**: `abcdefghijklmna`

**Payload**:
```json
{
  "i": "123456789",   // 用户ID
  "r": "0",           // 角色
  "ia": "0",          // 是否管理员
  "ai": "0",          // 地区ID
  "an": "",           // 地区名称
  "ex": 1767225600    // 过期时间戳
}
```

**Token 类型**:
| Token 类型 | 过期时间 | 说明 |
|------------|----------|------|
| token | 30 分钟 | 短期访问令牌 |
| refresh_token | 30 天 | 长期刷新令牌 |

## 六、密码安全规范

### 6.1 密码加密算法

```
password_hash = MD5(salt + "+_)(*&qwer" + password)
```

### 6.2 密码强度要求

- 长度：8-16位
- 字符要求：含大小写字母、数字、特殊字符

### 6.3 登录失败处理

- 失败5次以内：返回错误码 `100009` 或 `100010`
- 失败5次后：必须输入图形验证码
